Syscalls are pointers in low ram that can be used with the JSR instruction. Ex: jsr [1] executes the syscall MSG.

#   Name    Params       Returns    Destroys     Notes/Desc
0   YIELD   NONE         NONE       ABCXYZIJ EX    Context switches to the next task.
1   MSG     ABXYZ        NONE       ABCXYZIJ EX    Message PID A, with message B, giving parameters X, Y and Z
2   WFCOMP  A            BC         ABCXYZIJ EX    Waits for completion structure pointed to by A to be completed, returns the return code in B, and message in C.
3   GETMSG  NONE         ABXYZ      ABCXYZIJ EX    Gets a message, stores sending PID in A, message in B, and the parameters in XYZ. If no message is in queue it will set A = 0.
4   COMP    ABC          NONE       -              Signals a completion to a struct completion pointed to by C with code = A and message B.

Core System Structs & deffinitions:

struct {
	u16 code;    //A non 0 code indicates a completion
	u16 message;
} completion;

struct {
	u16 sender;
	u16 message;
	u16 p1, p2, p3;
} pending_msg;

struct {
	u16 *common_name;                      //offset 0
	u16 reserved [4];            
	
	struct task *sched_prev;               //offset 5
	struct task *sched_next;               //offset 6
	
	u16 saved_sp;                          //offset 7
	
	u8 first_msg_index;                    //offset 8 upper octet
	u8 msg_qty;                            //offset 8 lower octet
	
	struct pending_msg messages[4];        //offset 9
	
	u16 reserved.                          //offset 29.
	
	u16 stdout_process;                    //offset 30
	u16 stdin_process;                     //offset 31
	
} task;
